# Quickarte — Founding Product Brief

## Mission
Build a premium, minimalist, mobile-first commerce operating system for restaurants and local merchants — significantly better than Feady's and similar tools.

## Context
Feady's (feadys.com) is a Morocco-based, French-language SaaS/mobile platform for small businesses. It helps restaurants, cafés, hotels, and local merchants create digital menus, online stores, QR ordering, simple websites, and basic payment workflows. Positioned as an all-in-one commerce tool for Francophone markets.

## Thesis
We do NOT want "another Feady's." We want a much better product — radically simpler, more premium, easier to use, better designed, more trustworthy, more operationally powerful under the hood, but with a very simple minimalist frontend.

## Philosophy
- Powerful backend, very simple UI
- Opinionated workflows over settings
- Progressive disclosure
- Almost zero learning curve
- Elegant defaults, strong taste, very little clutter

## Design Taste
Clean, boxy, sharp brutalist aesthetic. Premium, calm, sharp, minimal, architectural, modern, high-contrast, serious. NOT playful, bubbly, generic SaaS, cluttered, or cheap POS.

Visual inspiration: Linear × Vercel × Stripe × Notion — adapted to restaurants and local commerce.

## Core User
Non-technical business owner managing from a phone with almost no training.

## Primary Direction
Start with restaurants / cafés / hospitality as wedge. Possibly expand to local retail later.

## Product Principles
- Create before configure
- Instant first value
- Progressive complexity
- One-screen-first design
- No settings jungle, no feature soup
- Every screen useful immediately
- Surface only what matters now
- Workflows outcome-oriented, not system-oriented

## MVP Scope

### Customer-facing
- Digital menu / storefront
- QR menu page
- Online ordering + checkout
- Reservation request or table ordering
- Simple branded mini website

### Merchant-facing
- Onboarding wizard
- Catalog manager
- Order manager
- Simple analytics
- Payment configuration
- Business profile
- QR generation
- Basic promotions

### Backend Foundation
- Robust product/order/customer/business models
- Role-based access
- Audit logs
- Event tracking
- Localization-ready architecture

## v1 Non-Goals
- Loyalty programs
- Advanced CRM
- Multi-warehouse
- Deep accounting
- Marketplace features
- Giant website builder
- Marketing automation
- Enterprise features

## Top-Level Merchant Navigation
Home · Catalog · Orders · Customers · Store

## Backend Domain Modules
Identity · Business · Catalog · Ordering · Payments · Growth · Channels · Insights
